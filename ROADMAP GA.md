# Graham Analyzer — Roadmap

> «The investor's chief problem — and even his worst enemy — is likely to be himself.»

> — Benjamin Graham, *The Intelligent Investor*

Чек-лист развития проекта. Структурирован по приоритетам и блокам:

сначала — то, что усиливает ядро (анализ по Грэму), затем — расширения

(торговля, портфель, макро, экосистема). В каждом блоке пункты упорядочены

от «минимально жизнеспособной версии» к «зрелой».

Легенда статусов:

- `[ ]` — не начато

## 1. Анализатор Грэма (ядро) — ближайший спринт

### 1.1 Справедливая стоимость акции

- [ ] **Graham Number** — «дешёвая» классическая формула:

`√(22.5 × EPS × BVPS)` (потолок 15·P/E × 1.5·P/B).

Считать на каждый отчёт, чтобы видеть динамику.

- [ ] **Graham Formula (revised, 1962)** — учёт темпов роста и доходности AAA-облигаций:

`V = EPS × (8.5 + 2g) × 4.4 / Y`, где `g` — ожидаемый темп роста прибыли (5-10 лет),

`Y` — доходность 10Y ОФЗ (RGBI proxy). Ставка-аналог bond yield AAA из оригинала.

- [ ] **Net Current Asset Value (NCAV / Net-Net)** — самая консервативная оценка:

`(Current Assets − Total Liabilities) / Shares`. Покупка ≤ ⅔ NCAV — классика

раннего Грэма, до сих пор работает на small-caps.

- [ ] **Earnings Power Value (EPV, Bruce Greenwald)** — `Normalized Earnings / WACC`,

без премии за рост. Хороший якорь для зрелых компаний.

- [ ] **Двухступенчатая DCF** для контрольной проверки (5 лет высокого роста + терминальная стоимость).

- [ ] **Reverse DCF** — обратная задача: «какой рост уже заложен в текущую цену?».

Сильный инструмент для отсеивания переоценённых историй.

- [ ] **Dividend Discount Model** (Gordon Growth) — для дивидендных историй вроде МТС, Сургутнефтегаза-преф.

### 1.2 Семь критериев защитного инвестора (`The Intelligent Investor`, гл. 14)

Реализовать в виде «светофора» (зелёный / жёлтый / красный) на странице компании:

1. [ ] **Adequate Size** — выручка ≥ N млрд ₽ (для РФ-рынка адаптировать порог: ~50 млрд?)

2. [ ] **Strong Financial Condition** — Current Ratio ≥ 2.0, LT debt ≤ Working Capital

3. [ ] **Earnings Stability** — положительная прибыль 10 лет подряд

4. [ ] **Dividend Record** — непрерывные выплаты ≥ 20 лет (для РФ адаптировать до 7-10 — не так много историй)

5. [ ] **Earnings Growth** — рост EPS ≥ ⅓ за 10 лет (3-летние средние на концах)

6. [ ] **Moderate P/E** — текущий P/E ≤ 15 (от 3-летнего среднего EPS)

7. [ ] **Moderate P/B** — P/E × P/B ≤ 22.5 (Graham number в форме множителя)

### 1.3 График справедливой/балансовой/рыночной цены

- [ ] Линейный график на странице компании: рыночная цена (MOEX дневные котировки),

Graham Number, BVPS, NCAV — все в одной плоскости по годам.

- [ ] Заливка зоны «Margin of Safety» (например, ниже 0.7 × Graham Number).

- [ ] Тултипы с источником (какой отчёт даёт BVPS, какой курс использован).

- [ ] Toggle-переключатели «показать/скрыть» каждую линию.

### 1.4 Дополнительные скоринговые модели (комплемент к Грэму)

Современные академические скоринги отлично дополняют классику:

- [ ] **Piotroski F-Score** (0–9) — качество фундамента (рост ROA, маржа, ликвидность).

Грэм + Piotroski показывают на бэктестах премию ~7-9% годовых.

- [ ] **Altman Z-Score** — риск банкротства. Очень полезно для проблемных эмитентов

(Мечел, AFK Sistema в трудные годы).
- [ ] **Beneish M-Score** — детектор «творческой бухгалтерии». Для российского рынка

особенно актуально (помним кейс TMK / Мечел).

- [ ] **Sloan Accruals Ratio** — насколько прибыль подтверждена денежным потоком.

- [ ] **Magic Formula (Greenblatt)** — ранжирование по EBIT/EV + ROIC.

## 2. Качество данных и операции с отчётами

- [ ] **e-disclosure.ru** как источник PDF — автозагрузка по ИНН/тикеру.

- [ ] **Тинькофф Pulse / Smart-Lab парсинг** — кросс-проверка цифр, выявление ошибок LLM.

- [ ] **Версионирование отчётов** — если компания переиздала отчёт (RAS-2023 v2),

хранить обе версии, отмечать активную.

- [ ] **Аудит отчёта** — пометка «verified by analyst» с историей кто/когда правил.

- [ ] **Восстановление LTM из 3-летних средних** — Грэм требовал именно усреднения

по циклу, чтобы убрать «пиковые» прибыли (см. кейс с угольщиками 1929 г.).

- [ ] **Корректировки на разовые статьи** — отдельное поле «non-recurring items»,

чтобы считать «нормализованную» прибыль (Buffett's owner earnings).

- [ ] **Owner Earnings** = Net Income + D&A − Maintenance CapEx ± ΔWC. Один из лучших

прокси для свободного денежного потока.

- [ ] Сравнение портфельной структуры с Грэма-рекомендованной (50/50 акции/облигации).

### 3.3 Аналитика портфеля

- [ ] **Allocation** — по секторам, по странам резидентства, по валютам.

- [ ] **Risk metrics** — beta vs IMOEX, max drawdown, Sharpe (нужны исторические NAV).

- [ ] **Dividend calendar** — когда какой эмитент платит, сколько ожидается.

- [ ] **Tax preview** — НДФЛ 13/15%, ЛДВ (3 года), купоны без удержания у некоторых,

ИИС-3 (5+ лет).

- [ ] **Rebalancing assistant** — подсказывает «продать X, купить Y», чтобы вернуть веса.

### 3.4 Торговля

- [ ] Котировки в реальном времени (`MarketDataService.MarketDataStream`).

- [ ] Лимитные / рыночные ордера.

- [ ] **Stop-loss / take-profit** на уровне приложения (T-Invest не всегда поддерживает на акциях).

- [ ] **Position sizing** — встроенный калькулятор Kelly или fractional Kelly,

ограничение «не более N% портфеля в одной бумаге».

- [ ] **Trade journal** — лог сделок с тегами «по Грэму», «спекуляция», «ребаланс»,

ретроспективный анализ «какие правила работают».

- [ ] **Paper trading** — параллельный виртуальный счёт для тестирования стратегий

без денежного риска.

## 7. Дополнительные идеи (из «школы Грэма-Додда»)

Это то, что сам бы добавил, исходя из духа Грэма и Додда и реалий российского рынка:

### 7.1 Качественный, а не только количественный анализ

В *Security Analysis* (1934, гл. 1-3) Грэм и Додд жёстко разделяли «инвестиции» и

«спекуляции». Чисто количественный фильтр не отделит хорошую компанию от

проблемной — нужен **qualitative score**:

- [ ] **Экономический ров** (moat) — checkbox-форма: бренд, сетевой эффект, эффект масштаба, регуляторика, переключающие издержки.

- [ ] **Качество менеджмента** — длительность работы CEO, его доля в акциях,

record of capital allocation.

- [ ] **Прозрачность** — публикует ли годовой отчёт на английском, проводит ли

conference calls, отвечает ли IR.

- [ ] **Корпоративное управление** — состав совета директоров, доля независимых,

история сделок с заинтересованностью.

### 7.2 Cyclically-Adjusted P/E (CAPE / Shiller P/E)

- [ ] Скользящие 10-летние средние EPS с поправкой на инфляцию (CPI Росстата).

Сравнение CAPE с историческим средним по эмитенту и по отрасли —

классический инструмент идентификации циклических пиков.

### 7.3 Спред «доходность акции − доходность облигации»

- [ ] Earnings Yield (E/P) минус доходность 10Y ОФЗ (RGBI). Грэм считал, что
премия должна быть ≥ 0 для покупки акции. На графике портфеля или

отдельной макро-странице.

### 7.4 Margin of Safety dashboard

- [ ] Расчёт «запаса прочности» в процентах для каждой бумаги в портфеле и

watchlist'а (`(Intrinsic Value − Price) / Intrinsic Value`). Цветовая

индикация: ≥ 33% — зелёный (Graham рекомендовал минимум треть).

### 7.5 Стресс-тесты

- [ ] **Что будет с компанией, если ставка ЦБ +5 п.п.?** — пересчёт P/E через

изменение Y в формуле Graham-1962 и cost of debt в DCF.

- [ ] **Что если выручка −20%?** — пересчёт ROE, debt covenants.

- [ ] **Сценарий «санкционный шок»** — отдельный профиль для экспортёров.

### 7.6 Облигации и квази-фонды

Грэм всегда говорил про **balanced portfolio** (50/50 акции/облигации):

- [ ] Раздел «Облигации» — ОФЗ, корпораты, флоатеры, ИЦБ. YTM, дюрация, реальная доходность.

- [ ] **БПИФы** — TBRU, TGLD и пр. — как proxy для ребалансировки.

### 7.7 Поведенческий учёт

- [ ] **Investment journal** — текстовое поле к каждой сделке: «почему купил»,

«при каком сценарии продам». Через год сверка с фактом — лучшая защита от

систематических ошибок (loss aversion, overconfidence). Это прямой совет

Грэма из эпиграфа выше.

- [ ] **Cooling-off period** — настройка «отложить покупку на 24 часа» для

ордеров выше N₽. Отсекает импульсивные сделки.

### 7.8 Образовательный слой

- [ ] **Glossary** — каждая метрика кликабельна, открывается popup с цитатой

из Грэма / Додда / Buffett'а и формулой.

- [ ] **Tutorial** — пошаговое онбординг-ревью первой компании пользователя.

- [ ] **«Проверь себя»** — мини-квизы по главам *The Intelligent Investor*.
### 7.9 Социальное / комьюнити (опционально, поздняя стадия)

- [ ] Публичные watchlist'ы и журналы (по согласию пользователя).

- [ ] Лидерборд по доходности портфеля относительно IMOEX.

- [ ] Анонимные «голосования» — насколько компания соответствует критериям Грэма

(краудсорсинг качественной оценки).

### 7.10 AI-помощник

- [ ] **Chat-ассистент по компании** — RAG поверх отчётов: «расскажи про долговую

нагрузку Газпрома за последние 5 лет», «найди non-recurring items в отчёте Лукойла за 2023».

- [ ] **Авто-резюме отчёта** — 5 буллетов на странице каждого отчёта (LLM-сгенерированные,

с ссылками на конкретные числа).

- [ ] **Аномалии** — модель помечает «странные» движения (резкий скачок D&A,

необычно высокий accruals ratio) на ревью аналитика.

## 8. Очерёдность (мой совет)

Если позволит время и силы, имеет смысл двигаться так:

1. **Закрыть ядро Грэма (1.1 + 1.2 + 1.3)** — без этого бренд «Graham Analyzer»

не оправдан. После этого продукт уже сам себя продаёт.

2. **Скоринги Piotroski / Altman / Beneish (1.4)** — дёшево по реализации,

очень дорого по ценности для пользователя.

3. **Макро-ставка и RGBI (4.1)** — потому что без неё Graham-1962 формула

считает в вакууме.

4. **Качественный скор (7.1) + Margin of Safety dashboard (7.4)** —

это превращает калькулятор в инвестиционного советника.

5. **T-Invest интеграция (3)** — большой кусок, но уже после того, как ядро

работает на «бумажных» портфелях.

6. **Стресс-тесты, бэктесты, journal (7.5, 7.7)** — для продвинутых

пользователей и retention.

7. **Социалка и обучение (7.8, 7.9)** — последним, когда есть аудитория.

## 9. Принципы (не забывать)
- **Инвариант валюты**: все денежные поля отчёта хранятся в `report.currency`,

конвертация в рубли — централизованно в `calc_multipliers`. Не плодить дубли.

- **Best-effort внешних API**: MOEX, ЦБ РФ, T-Invest, OpenAI. Любой из них

упадёт — приложение должно работать в degraded mode, не падать с 502.

- **Идемпотентность миграций и upsert'ов**: повторный запуск не должен

ломать данные.

- **Грэм-приоритет**: при конфликте «красиво в UI» vs «честно по Грэму» —

всегда честно. Лучше показать жёлтый светофор и объяснение, чем сгладить картинку.
